THE FOREST OPERATIONS TOOLKIT
For Learning and Development Managers, HR Directors, and Store Operations Leads
Version 1.0 — July 2026

=== WHAT THIS FOLDER CONTAINS ===

This folder is named "the-forest-operations-toolkit". It contains six sub-folders and this document. The six sub-folders are:

1. Zero-One-QUICK-START
2. Zero-Two-EDITING-GUIDES
3. Zero-Three-TOOLS-READY-TO-DEPLOY
4. Zero-Four-PRINTABLES
5. Zero-Five-TRAINING-MODULES
6. Zero-Six-SOURCE-FILES

=== WHO SHOULD USE THIS ===

You are a Learning and Development Manager, an HR Director, a Store Operations Lead, or a District Manager responsible for training dispensary staff across multiple locations. You need training materials that work immediately, can be edited without a programmer, and can be deployed on store tablets today.

=== HOW TO GET RUNNING IN TEN MINUTES ===

Step One. Open the folder named Zero-One-QUICK-START.
Step Two. Open the file named "Zero-Minute-Quick-Start.html" in your web browser.
Step Three. Follow the on-screen instructions.

That is it. You are running.

=== THE TWO TOOLS ===

Tool One. Medical Patient Concierge.
Location: Zero-Three-TOOLS-READY-TO-DEPLOY/Zero-One-Medical-Patient-Concierge.html
What it does: A six-step questionnaire that guides staff to recommend the right cannabis product for each customer. It generates a printable recommendation card with staff talking points and dosage guidance.
Who uses it: Every budtender and consultation associate at every register and consultation desk.
How to deploy: Copy the file to any tablet, computer, or phone. Double-click to open. No internet required. No installation.

Tool Two. SOP and Training Engine.
Location: Zero-Three-TOOLS-READY-TO-DEPLOY/Zero-Two-SOP-Training-Engine.html
What it does: Ten pre-built dispensary scenarios. Each scenario generates a complete training package: a step-by-step standard operating procedure, an Ohio compliance check, a two-minute manager training script, and three quiz questions.
Who uses it: Store Managers and Lead Associates during pre-shift huddles. New hires during onboarding.
How to deploy: Copy the file to any tablet, computer, or phone. Double-click to open. No internet required. No installation.

=== HOW TO EDIT THE TOOLS ===

You do not need to know how to code. You do not need a programmer. You need a text editor and the ability to follow numbered steps.

To edit the products in the Medical Patient Concierge:
Open the file named Zero-Two-EDITING-GUIDES/How-to-Edit-the-Medical-Concierge.txt
Follow every step in order. Do not skip steps.

To edit the scenarios in the SOP Training Engine:
Open the file named Zero-Two-EDITING-GUIDES/How-to-Edit-the-SOP-Engine.txt
Follow every step in order. Do not skip steps.

=== HOW TO PRINT MATERIALS ===

Open any HTML file in your web browser. Press the Control key and the letter P at the same time. On a Mac, press the Command key and the letter P at the same time. Select your printer. Click Print.

For pre-built PDFs, open the folder named Zero-Four-PRINTABLES. Double-click any PDF file. It will open in your PDF viewer. Press Control-P or Command-P to print.

=== HOW TO DEPLOY ON STORE TABLETS ===

Option A. Email the HTML files to each store. The store manager downloads them to the store tablet. Double-click to open. Done.

Option B. Copy the HTML files to a USB drive. Hand the USB drive to each store manager. The store manager copies the files from the USB drive to the store tablet. Double-click to open. Done.

Option C. Upload the HTML files to a shared company drive such as Google Drive, OneDrive, or Dropbox. Share the link with all store managers. Store managers download the files to their tablets. Double-click to open. Done.

No app store. No IT ticket. No corporate approval. The files work offline.

=== HOW TO UPDATE THE TOOLS ===

When you edit a tool, save the edited file with a new version number in the filename. For example: Medical-Patient-Concierge-v2.html. Email the new version to all stores. Ask store managers to replace the old file with the new file. The old file can be deleted.

=== SUPPORT ===

If a file will not open, right-click the file. Select Open With. Choose a web browser such as Chrome, Edge, Safari, or Firefox. Do not choose a text editor unless you are intentionally editing the file.

If the print button does not work, press Control-P or Command-P manually.

If a file looks wrong on a phone, use a tablet or computer instead. These tools are designed for tablets and computers.
